## Общая формулировка задания
Необходимо реализовать ETL пайплайн (В данном случае приложение на Scala, Java или Python,
которое загружает исходные данные, трансформирует их, а затем выгружает) 
для выгрузки данных прогнозов погоды.

Перед формулировкой конкретных заданий, предупрежу, что данное тестовое задание я взял за основу практики
программирования на языке **Scala**, поэтому хоть и будет дана задача наивной реализации, я подошел к выполнению
более "серьезно" для укрепления навыков. \
Также для каждого этапа (задания) будет выделена соответсвующая ветвь (git branch), ради наработки практики в git'е

## Задание 1: Наивная реализация

Реализовать приложение, которое будет делать следующее:
1. считывать json в формате API open-meteo, json взять из запроса в
   "Пример запроса данных по API open-meteo"
2. Выполнять трансформации данных, которые на Ваш взгляд необходимы для
   получения данных в формате для выгрузки - т.е в формате, как в таблице
   "Итоговая таблица" из **приложения**.
3. Сохранить данные в csv. Поля, необходимые в csv и их описание представлены в
   таблице "Итоговая таблица" из **приложения**.

### Реализация

Были реализованы:
* Класс OpenMeteoClient - отвечает за отправку GET запроса и формирование JSON объекта/файла для последующего преобразования в DataFrame
* Класс OpenMeteoView - инициализируется с помощью датафреймов данных и их ед. измерения, 
содержит в себе методы преобразований и агрегаций полей объектов данного класса 
* Класс и объект-компаньон OpenMeteo - содержат необходимые для работы пайплайна константы и
методы преобразований json объектов в DataFrame, сохранения их в формат CSV и исполнения ETL пайплайна

## Задание 2*: Вставка в базу данных

Реализовать возможность вставки в базу данных в формате выходной таблицы из задания 1 (базу можно выбрать любую).
1. Реализовать базу данных в виде docker-compose скрипта + скрипты для создания
   таблицы и базы данных.
2. Предусмотреть случай вставки дубликатов.

### Реализация

Были реализованы:
* Docker-compose файл, совместно с init.sql скриптом, которые поднимают PostgreSQL БД и создают схемы таблиц соответствующих требованиям выгружаемых таблиц
* Метод, выполняющий вставку датафрейма (таблицы) в указанную таблицу JDBC БД без дубликатов 4 возможными стратегиями (определяется с помощью соответствующего аргумента приложения):
  * OverwriteAll - Полностью перезаписывает таблицу БД (полная очистка и последующая вставка)
  * OverwriteInterval - Перезапись таблицы происходит по выборке ключевых столбцов (очистка таблицы по выгружаемым ключам и последующая вставка по выборке)
  * UpsertConflict - Выполняет Upsert (insert + update) в таблицу с помощью предложения ON CONFLICT
  * Upsert - Выполняет Upsert в таблицу без использования предложения ON CONFLICT (для JDBC баз данных не имеющих логики ON CONFLICT)

2. Предусмотреть выгрузку в базу данных.
